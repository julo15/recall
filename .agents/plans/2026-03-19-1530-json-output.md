# Add --json output mode to recall CLI

## Goal

Add a `--json` flag so recall outputs structured JSON, enabling programmatic use from seshboard (Swift app) and other tools.

## Changes

### `recall/cli.py`

- Add `--json` argument (store_true)
- When `--json` is set:
  - Suppress all human-readable prints (reindex progress, "No matching sessions", etc.) by redirecting status messages to stderr
  - Output a JSON array to stdout with this shape:

```json
[
  {
    "agent": "claude",
    "session_id": "abc-123",
    "project": "/Users/foo/bar",
    "timestamp": 1710000000.0,
    "score": 0.359,
    "resume_cmd": "claude --resume abc-123",
    "text": "the matched text snippet"
  }
]
```

- When `--json` is set and no results: output `[]`
- Existing human-readable output unchanged when `--json` is not set

### That's it

No changes to search.py, index.py, embedding.py, or adapters. This is purely a CLI presentation concern.

## Seshboard integration (separate repo, later)

Seshboard will call `recall --json -n 10 "query"` via `Process()`, decode the JSON array, and display results. That work happens in the seshboard repo, not here.
