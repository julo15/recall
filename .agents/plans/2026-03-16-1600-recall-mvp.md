# Recall — Semantic Search Across Agent Sessions

## Goal
A CLI tool that lets you search across Claude, Codex, and Gemini conversation history using natural language. Returns matching sessions with context and resume commands.

## Example Usage
```bash
recall "upload profile image issue"
recall "that auth middleware refactor"
recall --agent claude "notification batching"
recall --reindex  # force full reindex
```

## Architecture

```
recall/
├── recall/
│   ├── __init__.py
│   ├── cli.py              # CLI entrypoint (argparse or click)
│   ├── search.py           # Embedding + cosine similarity search
│   ├── index.py            # Index management (build, update, persist)
│   └── adapters/
│       ├── __init__.py
│       ├── base.py         # Base adapter protocol/ABC
│       ├── claude.py       # ~/.claude/history.jsonl parser
│       ├── codex.py        # ~/.codex/history.jsonl parser
│       └── gemini.py       # ~/.gemini/tmp/*/logs.json parser
├── pyproject.toml
└── README.md
```

## Data Model

Each indexed entry:
```python
@dataclass
class HistoryEntry:
    text: str           # The user's prompt
    agent: str          # "claude" | "codex" | "gemini"
    session_id: str     # Agent-specific session identifier
    project: str        # Project path or name
    timestamp: float    # Unix timestamp
```

## Index Storage

Location: `~/.recall/`
```
~/.recall/
├── embeddings.npy    # (N, D) numpy array of embedding vectors
├── metadata.jsonl    # One HistoryEntry per line, aligned by row index
└── cursors.json      # Per-adapter cursor tracking last-indexed position
```

## Implementation Plan

### Phase 1: Adapters
- [ ] Define `HistoryEntry` dataclass and base adapter protocol
- [ ] Claude adapter: parse `~/.claude/history.jsonl`
  - Each line has `display`, `sessionId`, `project`, `timestamp`
  - Resume command: `claude --resume <sessionId>`
- [ ] Codex adapter: parse `~/.codex/history.jsonl`
  - Each line has `text`, `session_id`, `ts`
  - No project field in index — derive from session file path in `~/.codex/sessions/`
  - Resume command: `codex --resume <session_id>`
- [ ] Gemini adapter: parse `~/.gemini/tmp/*/logs.json`
  - Each file is a JSON array; filter to `type: "user"` entries
  - Project path from `.project_root` file in same directory
  - Resume command: `gemini` (no built-in resume by ID — note this)

### Phase 2: Embedding + Indexing
- [ ] Choose local embedding model — `sentence-transformers/all-MiniLM-L6-v2`
  - 384-dim vectors, fast, no API key needed
  - ~80MB model, downloads once on first run
- [ ] Build index: load all adapters → collect new entries since cursor → embed → append
- [ ] Incremental update: each adapter tracks its own cursor
  - Claude/Codex: byte offset in JSONL file
  - Gemini: set of seen `(sessionId, messageId)` tuples or file mtime
- [ ] Persist index to `~/.recall/`

### Phase 3: Search
- [ ] Embed query using same model
- [ ] Cosine similarity against all stored embeddings
- [ ] Return top-N results (default 5)
- [ ] Deduplicate: if multiple prompts from same session match, show the best one
- [ ] Format output with agent, date, project, prompt snippet, and resume command

### Phase 4: CLI
- [ ] `recall <query>` — search (auto-indexes first if needed)
- [ ] `recall --reindex` — force full reindex
- [ ] `recall --agent <name>` — filter to one agent
- [ ] `recall --limit N` — control result count
- [ ] `recall --since <date>` — filter by date

### Phase 5: Polish
- [ ] `pyproject.toml` with `[project.scripts]` entry for `recall` command
- [ ] Install via `pip install -e .`
- [ ] Handle edge cases: missing history files, empty sessions, corrupted JSON lines
- [ ] Progress bar on first index build (tqdm)

## Decisions
- **Local model over API**: No API keys, works offline, fast enough for <10K entries
- **Prompt-level indexing** (not full conversation): Keeps index small, prompts are the best signal for "what was I working on"
- **Shared index across agents**: One search, all agents — tag results by source
- **No vector DB**: numpy + cosine similarity is plenty for this scale
